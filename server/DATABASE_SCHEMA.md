# MongoDB Database Schema Specification

This project utilizes MongoDB Atlas for storage. Below is a specification of every database collection, including schemas, data types, field constraints, relationships, and indexing patterns.

---

## 1. Admins Collection (`admins`)

Maintains admin login accounts. Designed for security and single admin constraints (though the schema supports multiple admins for future scale).

### Schema Specification
| Field Name | Mongoose Type | Validation Constraints | Description |
| :--- | :--- | :--- | :--- |
| `_id` | `ObjectId` | Auto-generated by MongoDB | Unique identifier |
| `email` | `String` | Required, unique, lowercase, trimmed, Regex Email format | Administrative login email |
| `password` | `String` | Required, length >= 8, `select: false` (hidden by default) | Bcrypt-hashed password |
| `resetPasswordToken` | `String` | Optional | Generated token for password reset |
| `resetPasswordExpire`| `Date` | Optional | Expiration timestamp of reset token |
| `createdAt` | `Date` | Generated automatically | Document creation timestamp |
| `updatedAt` | `Date` | Generated automatically | Document modification timestamp |

### Indexes
*   `{ email: 1 }` (Unique) - Ensures no duplicate admin emails can exist and speeds up lookup during login queries.

## 2. Products Collection (`products`)

Stores all items in the portfolio database. Supports descriptions, categories, specifications, search indexing, and a soft-delete status.

### Schema Specification
| Field Name | Mongoose Type | Validation Constraints | Description |
| :--- | :--- | :--- | :--- |
| `_id` | `ObjectId` | Auto-generated | Unique identifier |
| `name` | `String` | Required, unique, trimmed | Product name |
| `slug` | `String` | Unique, lowercase, automatic slugification | URL slug identifier |
| `shortDescription` | `String` | Required, max length: 300, trimmed | Quick description |
| `description` | `String` | Required, trimmed | Comprehensive specifications and features |
| `category` | `String` | Required, trimmed | Grouping category |
| `images` | `Array` (Sub-docs)| Default: `[]` | Array of Image objects (see sub-schema below) |
| `availableColors` | `Array` (String) | Default: `[]` | Colors options (e.g. `['Charcoal', 'Tan']`) |
| `availableSizes` | `Array` (String) | Default: `[]` | Size options (e.g. `['US 8', 'US 9']`) |
| `material` | `String` | Optional, trimmed | Raw materials |
| `features` | `Array` (String) | Default: `[]` | Highlights bullet-points |
| `isFeatured` | `Boolean` | Default: `false` | Highlighted on home/featured lists |
| `isActive` | `Boolean` | Default: `true` | Publicly browseable status |
| `isDeleted` | `Boolean` | Default: `false`, `select: false` | Soft-delete toggle flag |
| `createdAt` | `Date` | Auto-generated | Creation date |
| `updatedAt` | `Date` | Auto-generated | Last modification date |

#### Image Sub-Schema (`productImageSchema`)
*   `url` (String, required): Public image CDN URL (or local filesystem fallback url).
*   `public_id` (String, required, default: `local_placeholder`): Cloudinary reference ID for removal operations.

### Indexes
*   `{ slug: 1 }` (Unique) - Accelerates GET product detail lookups.
*   `{ category: 1 }` - Optimizes product category filtering.
*   `{ isFeatured: 1, isActive: 1, isDeleted: 1 }` - Compound query index for landing pages.
*   `{ name: 'text', description: 'text', category: 'text' }` - Compound full-text search index for customer searches.

## 3. Testimonials Collection (`testimonials`)

Maintains client reviews and customer feedback.

### Schema Specification
| Field Name | Mongoose Type | Validation Constraints | Description |
| :--- | :--- | :--- | :--- |
| `_id` | `ObjectId` | Auto-generated | Unique identifier |
| `customerName` | `String` | Required, trimmed | Customer full name |
| `message` | `String` | Required, trimmed | Review content |
| `source` | `String` | Required, Enum: `['WhatsApp', 'Instagram DM', 'Instagram Story', 'Text']` | Review source channel |
| `rating` | `Number` | Optional, Min: 1, Max: 5 | Numeric rating out of 5 |
| `isFeatured` | `Boolean` | Default: `false` | Highlighted testimonial status |
| `image` | `Object` (Sub-doc) | Optional | Customer profile avatar details (same as Image Schema) |
| `createdAt` | `Date` | Auto-generated | Creation timestamp |

### Indexes
*   `{ isFeatured: 1 }` - Speeds up landing page featured testimonials list.

## 5. Settings Collection (`settings`)

Stored singleton record containing global parameters. Static helpers prevent the creation of more than one settings entry.

### Schema Specification
| Field Name | Mongoose Type | Validation Constraints | Description |
| :--- | :--- | :--- | :--- |
| `_id` | `ObjectId` | Auto-generated | Unique identifier |
| `businessName` | `String` | Required, default: 'My Showcase Business', trimmed | Company/Showcase title |
| `phone` | `String` | Required, default: '+1234567890', trimmed | Core business contact line |
| `whatsapp` | `String` | Required, default: '+1234567890', trimmed | WhatsApp chat line link |
| `email` | `String` | Required, default: 'contact@mybusiness.com' | Business email |
| `instagram` | `String` | Default: Instagram url link, trimmed | Instagram page handle |
| `address` | `String` | Default: Address text, trimmed | Store physical address |
| `logo` | `Object` (Sub-doc) | Optional | Company logo image details (url, public_id) |
| `heroImage` | `Object` (Sub-doc) | Optional | Banner home screen graphic details |
| `aboutText` | `String` | Default: About details, trimmed | Company description block |
| `createdAt` | `Date` | Auto-generated | Settings entry creation date |
| `updatedAt` | `Date` | Auto-generated | Settings last update date |
